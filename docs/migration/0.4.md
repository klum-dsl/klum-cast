# Migrating to KlumCast 0.4

KlumCast 0.4 is the breaking transition release before 1.0. It requires Java 17 and supports Groovy 3, 4, and 5. Groovy
2.4 and Java 11 remain on KlumCast 0.3; they are not release or CI lanes for 0.4.

## Select dependencies by role

- Keep `klum-cast-annotations` on annotation/API classpaths. It is Groovy-free.
- Add `klum-cast-spi` only where custom checks, filters, or typed `@CheckBinding` declarations are compiled.
- Add `klum-cast-compile` only to Groovy compilation classpaths that should activate validation.
- Custom-check and compiler-plugin projects must explicitly add their matching Groovy compiler dependency. SPI metadata no
  longer selects one transitively.

Use `org.codehaus.groovy:groovy` for Groovy 3 and `org.apache.groovy:groovy` for Groovy 4 or 5. On the module path, the
matching module names are `org.codehaus.groovy` and `org.apache.groovy`, respectively. The KlumCast automatic module names
remain `com.blackbuild.klum.cast.annotations`, `com.blackbuild.klum.cast.spi`, and
`com.blackbuild.klum.cast.compiler`.

## Migrate custom checks

Implement the stateless `com.blackbuild.klum.cast.spi.Check` interface and return `Diagnostic` values from
`check(CheckContext)`. Prefer `@CheckBinding(MyCheck.class)` for typed co-located checks; use
`@KlumCastValidator("fully.qualified.Check")` when metadata and implementation must remain in separate dependency layers.
Expected annotation-use failures are diagnostics. Unexpected exceptions remain technical failures.

`com.blackbuild.klum.cast.checks.impl.KlumCastCheck`, `ValidationException`, legacy mutable invocation fields,
annotation-valued `@OneCheckMustMatch` branches, and the raw `KlumCastValidator.type()` bridge exist only for 0.4 source or
binary migration. Replace them before 1.0, when these bridges are removed.

## Verify the consumer build

Compile and test the consumer on Java 17 in each supported Groovy generation. Never reuse Groovy- or Spock-compiled test
or fixture output between generations. If the build uses named modules, change the Groovy `requires` clause at the Groovy
3/4 boundary. A Gradle and Maven reference fixture is kept under
`docs/implementation/fixtures/published-consumer`.
